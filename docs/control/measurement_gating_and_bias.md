# 量测门控与 Bias 可观测性

## 作用

当前 Quaternion EKF 不会无条件相信每一次加速度更新。

在真正融合之前，它会先判断：

“当前加速度方向看起来还像不像一个可信的重力方向观测。”

加速度计中可能混入：

- 线加速度
- 冲击
- 振动
- 其它非重力成分

## 残差与创新协方差

量测残差为：

```math
r = z - h(\hat x^-)
```

创新协方差为：

```math
S = H P^- H^T + R
```

它们的含义是：

- `r` 表示实际量测和预测量测差了多少
- `S` 表示这个残差本来应该有多大的不确定性

所以代码不能只看 `r` 的大小，而要看：

“当前残差相对于预期不确定性来说，是不是异常大。”

## 卡方统计量

代码中计算：

```math
\chi^2 = r^T S^{-1} r
```

它就是残差的马氏距离平方。

含义可以简单理解成：

- `chi2` 小：当前量测和预测比较一致
- `chi2` 中等：量测还能用，但需要降权
- `chi2` 很大：量测比较可疑，可能要拒绝

在本工程中：

- 很小的 `chi2` 会帮助滤波器进入收敛状态
- 中等的 `chi2` 会通过 `AdaptiveGainScale` 缩小增益
- 过大的 `chi2` 会触发“只预测、不更新”之类的处理

## 为什么 bias 需要额外处理

状态里包含了 gyro bias，但重力对不同轴的 bias 可观测性并不一样。

核心直觉是：

- 如果某个轴接近重力方向，那么绕这个轴转动时，重力投影变化很小
- 这时加速度计对该轴 bias 的辨识能力就弱
- 如果某个轴和重力方向近似垂直，这个轴的 bias 会更容易被观测到

工程中会根据“预测重力方向和机体坐标轴的夹角”，对 `K` 中 bias 那几行再做一次缩放。

## 工程效果

当前实现结合了三件事：

- 残差一致性检验
- 整体自适应增益缩放
- bias 方向性增益缩放

这样做的结果是：

- 静止或低动态时，姿态和 bias 可以较稳定地被修正
- 剧烈运动时，不容易把坏量测硬塞进滤波器
- 可观测性较差的 bias 方向会更新得更保守
